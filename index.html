install Python

Django is a Python web framework, thus requiring Python to be installed on your machine.
To install Python on your machine go to Python downloads
After installation, open the command prompt and check the Python version.

...\> py --version

About pip

pip is a package manager for Python and is included by default with the Python installer. It helps to install and uninstall Python packages (such as Django!). After installation of Python, open the command prompt and check the pip version.

...\> pip --version

to upgrade pip
...\> Python -m pip install --upgrade pip

Virtualenv vs venv

venv is a package shipped directly with python 3. So you don't need to pip install anything. virtualenv instead is an independent library and can be install with pip. They solve the same problem and work in a very similar manner. If you use python3 I suggest to avoid an "extra" dependancy and just stick with venv.

Setting up a virtual environment

It is best practice to provide a dedicated environment for each Django project you create. To create a virtual environment for your project, open a new command prompt, navigate to the folder where you want to create your project for 'instance djprojects' and then enter the following:

C:\Users\NaderMekky\Desktop\djprojects> py -m venv project-name

This will create a folder called ‘project-name’ if it does not already exist and setup the virtual environment. To activate the environment, run:

C:\Users\NaderMekky\Desktop\djprojects\project-name\Scripts\activate.bat

The virtual environment will be activated and you’ll see “(project-name)” next to the command prompt to designate that. Each time you start a new command prompt, you’ll need to activate the environment again.

To exit your virtualenv just type “deactivate”

Structure Django Project

Inside your virtual environment folder create new folder and name it ‘src’ for instance and then inside ‘src’ folder create new folder and name it ‘frontend’ for instance.

Installing Django

Django can be installed easily using pip within your virtual environment. In the command prompt, ensure your virtual environment is active, and execute the following command:

(project-name) C:\Users\NaderMekky\Desktop\djprojects\projectname\src> py -m pip install Django
After the installation has completed, you can verify your Django installation by executing
(project-name) C:\Users\NaderMekky\Desktop\djprojects\projectname\src>django-admin --version

Creating Project

From the command line, cd into project-name directory, then run the following command:

(project-name) C:\Users\NaderMekky\Desktop\djprojects\projectname\src> django-admin startproject project .

Run the Development Server

Let’s verify your Django project works. cd into the outer project directory, if you haven’t already, and run the following commands:

(project-name) C:\Users\NaderMekky\Desktop\djprojects\projectname\src> python manage.py runserver

Unapplied Migration(s)

If you get this warning :

You have 18 unapplied migration(s). Your project may not work properly until you apply the migrations for app(s): admin, auth, contenttypes, sessions.
so you need to stop the server CTRL+C, and run the following commands:
(project-name) C:\Users\NaderMekky\Desktop\djprojects\projectname\src> python manage.py migrate
to apply the migration(s), then run the server again.

Creating your first app

To create your app, make sure you’re in the same directory as manage.py and type this command:

(project-name) C:\Users\NaderMekky\Desktop\djprojects\projectname\src> py manage.py startapp app1

Include app1 in our project

To include the app in our project, we need to add a reference to its configuration class in the INSTALLED_APPS setting. The App1Config class is in the app1/apps.py file, so its dotted path is 'app1.apps.App1Config'. Edit the project/settings.py file and add that dotted path to the INSTALLED_APPS setting. It’ll look like this:

project/settings.py
                    INSTALLED_APPS = [
                    'app1.apps.App1Config',
                    'django.contrib.admin',
                    'django.contrib.auth',
                    'django.contrib.contenttypes',
                    'django.contrib.sessions',
                    'django.contrib.messages',
                    'django.contrib.staticfiles',
                ]

Write your first view

Let’s write the first view. Open the file app1/views.py and put the following Python code in it:

app1/views.py
                    from django.shortcuts import render
                    from django.http import HttpResponse
                    
                    def index(request):
                        return HttpResponse("Hello, world. You're at the Apps1 index.")

Call the view

This is the simplest view possible in Django. To call the view, we need to map it to a URL - and for this we need a URLconf. To create a URLconf in the app1 directory, create a file called urls.py . Your app directory should now look like:

app1/
                    __init__.py
                    admin.py
                    apps.py
                    migrations/
                        __init__.py
                    models.py
                    tests.py
                    urls.py
                    views.py

In the app1/urls.py file include the following code:

app1/urls.py
                    from django.urls import path
                    from . import views
                    
                    urlpatterns = [
                        path('', views.index, name='index'),
                    ]

The next step is to point the root URLconf at the app1.urls module. In project/urls.py, add an import for django.urls.include and insert an include() in the urlpatterns list, so you have:

project/urls.py
                    from django.contrib import admin
                    from django.urls import include, path
                    
                    urlpatterns = [
                        path('app1/', include('app1.urls')),
                        path('admin/', admin.site.urls),
                    ]

Now you can activate the virtual environment

C:\Users\NaderMekky\Desktop\djprojects\projectname\src >Scripts\activate.bat

and run the server
(project-name) C:\Users\NaderMekky\Desktop\djprojects\projectname\src> python manage.py runserver
then Go to http://localhost:8000/app1/ in your browser, and you should see the text “Hello, world. You’re at the app1 index.”, which you defined in the index view.
